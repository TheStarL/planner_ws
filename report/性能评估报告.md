## 2. 系统架构

```
  感知 (Perception)          规划 (Planning)                     控制 (Control)
 ┌────────────────┐   ┌───────────────────────────────┐   ┌────────────────────┐
 │  栅格/盒式障碍   │   │ 前端: Kinodynamic A*           │   │ PD + 前馈 跟踪控制器 │
 │  地图 (已知世界) │──▶│   运动基元搜索, 满足 v/a 约束    │──▶│ 二阶积分器机器人模型 │──▶ /odom
 │                │   │ 后端: B样条 + 梯度优化(ESDF/EGO)│   │ 跟踪优化后的时间轨迹 │
 └────────────────┘   │ 时间分配: 按弧长/巡航速度       │   └────────────────────┘
                      └───────────────────────────────┘
```

- **前端（Kinodynamic A*）**：在 `(位置, 速度)` 状态空间中以离散加速度为控制量做运动基元扩展，
  代价 `g = Σ(Δt + ρ‖a‖²Δt)`，启发项为到目标的时间下界，保证生成的路径**动力学可行**（满足最大速度/加速度）。
- **后端（B样条优化）**：将前端路径作为均匀三次 B样条的控制点，最小化
  `J = λ_s·J_smooth + λ_c·J_collision`。其中平滑项为控制点二阶差分能量，碰撞项把轨迹推离障碍。
  - **ESDF 方式（Fast-Planner）**：预先构建欧氏符号距离场（ESDF），碰撞梯度由距离场插值得到。
  - **EGO 方式（EGO-Planner）**：**不构建 ESDF**，对碰撞控制点即时生成锚点 `p` 与排斥方向 `v`，
    距离估计 `d = (q − p)·v`，碰撞梯度 `−2λ_c(d_safe − d)·v`，解析求得、无需任何场查询。
- **控制**：对时间参数化后的轨迹做 PD + 加速度前馈跟踪，二阶积分器模型，带加速度/速度限幅。

## 5. 子任务②：后端 B样条 + ESDF 梯度优化前后对比（平滑性 / 安全性）

`bspline_backend_demo` 实现的是**真正的 Fast-Planner 后端**：

1. 构建真实 **ESDF（欧氏符号距离场）**：在 0.10 m 分辨率栅格上，逐体素计算到最近膨胀障碍盒的
   符号距离（盒体 SDF 的并集取 min），本场景栅格 121×101×33 ≈ **40.3 万体素**，构建耗时约 233 ms。
2. 以前端路径为均匀三次 B样条控制点，做**梯度下降优化**，代价函数
   `J = λ_s·J_smooth + λ_c·J_collision + λ_f·J_feasibility`：
   - 平滑项：控制点二阶差分能量（elastic）；
   - **碰撞项：距离场 `d` 与梯度 `∇d` 由 ESDF 三线性插值 + 中心差分得到**，当 `d<d_safe` 时
     施加 `−2λ_c(d_safe−d)∇d` 把曲线推离障碍；
   - 可行性项：由 B样条速度/加速度控制点惩罚超过 v_max / a_max 的部分。
   （每步对控制点位移做幅值限制以保证数值稳定；优化耗时约 3 ms / 500 次迭代。）

优化前（前端原始动力学路径）与优化后（ESDF 优化的 B样条）对比：

| 指标 | 优化前 | 优化后 | 变化 |
|---|---|---|---|
| **平滑度代价**（二阶差分能量，越小越平滑） | 0.5376 | **0.00397** | **降低约 135 倍** |
| **最小安全距离 (m)** | 0.286 | **0.543** | **+90%（更安全，逼近 d_safe=0.55）** |
| 轨迹长度 (m) | 13.087 | 13.317 | +1.8%（略增，换取安全裕度） |
| 最大速度 (m/s) | 2.993 | 3.062 | ≈ 限制 3.0 |
| **最大加速度 (m/s²)** | 6.403 | **2.359** | 由超限 6.4 压回逼近限制 2.0 |
| 是否无碰撞 | true | **true** | 保持安全 |

> 速度/加速度由 **B样条导数控制点**精确求得（非密采样有限差分），是轨迹的真实运动学上界。

**结论**：ESDF 梯度优化在几乎不增加路径长度的前提下，**同时改善了三项指标**——
平滑度代价下降约 2 个数量级、最小障碍距离提升约 90%（0.286→0.543 m）、最大加速度由 6.4 压到 2.36 m/s²
（接近动力学限制），并保持全程无碰撞。这验证了 Fast-Planner 后端「平滑性 + 安全性 + 动力学可行性」
三目标联合优化的有效性。

## 6. 子任务③：EGO-Planner vs ESDF 计算效率对比（核心）

在**同一前端路径**上分别运行 ESDF 后端与 EGO 后端，唯一区别是碰撞梯度来源。
`ego_planner_demo` 单次运行结果：

| 后端 | 距离场构建 (ms) | 优化 (ms) | **总耗时 (ms)** | 长度 (m) | 平滑度 | 最小距离 (m) | 无碰撞 |
|---|---|---|---|---|---|---|---|
| ESDF (Fast-Planner) | 112.41 | 0.28 | **112.69** | 13.065 | 0.0042 | 0.310 | true |
| **EGO (ESDF-free)** | **0.00** | 2.15 | **2.15** | 13.040 | 0.0042 | 0.305 | true |

- ESDF 栅格规模：101×84×28 = **237,552 个体素**，每次规划都需重建。
- **EGO 总耗时加速比：约 52×**（单次场景）。

> 注：EGO 的优化耗时（2.15 ms）反而略高于 ESDF 优化（0.28 ms），因为 EGO 每次查询需遍历障碍物求最近锚点，
> 而 ESDF 优化只是一次廉价的栅格插值。但 **ESDF 的距离场构建（112 ms）才是真正的瓶颈**，
> EGO 完全省去了这一步——这正是 EGO-Planner 论文的核心论点：避免维护全局 ESDF 即可大幅提速。

**批量统计（30 组随机地图，见第 8 节）：EGO 平均总耗时 2.47 ms，ESDF 平均 107.8 ms，平均加速比 ≈ 44.4×。**
两者轨迹长度、平滑度、安全距离基本一致，说明 EGO **在不损失轨迹质量的前提下显著提升计算效率**。

## 7. 子任务④：感知-规划-控制全链路闭环（含动态避障）

`closed_loop_demo` 串联 地图(感知) → Kinodynamic A* 前端 → EGO B样条后端 → 时间分配 →
PD+前馈跟踪控制器，驱动机器人模型沿优化轨迹运动。支持两种模式（ROS 参数 `dynamic` / `disturbance` 切换）：

- **静态基线模式**（`dynamic:=false disturbance:=false`）：理想二阶积分器，单次规划。
- **动态滚动重规划模式**（默认）：见第 10 节升级——加入移动障碍、毫秒级局部重规划与执行器噪声/扰动。

| 指标 | 静态基线 | **动态+噪声模式** |
|---|---|---|
| 模式 | dynamic=off, disturbance=off | dynamic=on, disturbance=on |
| 到达目标时间 | 9.34 s | 8.51 s |
| 滚动重规划次数 | 0 | **37** |
| **RMS 跟踪误差** | 0.133 m | **0.129 m** |
| 最大跟踪误差 | 0.328 m | 0.365 m |
| 执行轨迹无碰撞 | true | **true（0 步碰撞）** |

**结论**：闭环系统在两种模式下均稳定收敛到目标，RMS 跟踪误差约 0.13 m、全程无碰撞。
即便在移动障碍 + 控制延迟 + 过程噪声 + 恒定扰动下，依靠每 0.25 s 一次的毫秒级局部重规划（共 37 次），
机器人仍 0 碰撞到达目标，验证了“感知-规划-控制”全链路在动态环境下的连通性与鲁棒性。

## 9. 仿真环境的改进与剩余局限

初版仿真环境较为简单（已知真值盒障碍、全静态、单次规划、理想点质量机器人）。本版本针对性升级：

**已实现的升级（`closed_loop_demo` 动态模式，默认开启）**
1. **动态障碍**：2 个移动障碍在场内匀速运动并边界反弹（RViz 中以**橙色**区别于静态红色障碍）。
2. **滚动重规划（receding-horizon）**：初次全规划后，每 0.25 s 用 EGO 后端对当前 B样条**热启动局部重优化**
   （约 1 ms / 次），并把跟踪时间**重锚定**到机器人当前进度，实现边走边躲的实时局部重规划——
   这正是 EGO/Fast-Planner 这类局部规划器的真实用途。本场景全程 37 次重规划，0 碰撞到达。
3. **更真实的执行器**：控制指令带**一步延迟** + **高斯过程噪声（σ=0.35 m/s²）** + **恒定扰动**，
   考核跟踪鲁棒性；结果 RMS 误差仍仅 0.129 m、0 碰撞。

**已实现的升级（`mapping_demo`：有限视场传感器 + 增量占据/ESDF 建图）**

为让“感知”不再作弊，新增 `mapping_demo` 节点：真值世界对规划器**不可见**，机器人靠一个
**有限距离 + 有限视场角的激光式传感器**边走边建图：

1. **传感器模型**：以机器人朝向为中心发射扇形射线（默认 range=3.5 m，FOV=120°），
   射线遇到真值障碍即停（**产生遮挡阴影**），命中点写入占据栅格。
2. **增量占据栅格**：三态（UNKNOWN / FREE / OCC），分辨率 0.20 m；未知区域**乐观地当作自由**。
3. **增量 ESDF**：用 chamfer 距离变换从已知 OCC 栅格重建 2D ESDF，供 EGO 后端读取碰撞梯度。
4. **只在已知地图上规划**：前端 Kino A* 与 EGO 后端仅使用已知栅格；当传感器**新发现**的障碍挡住
   前视轨迹（look-ahead）时，触发前端全重规划绕行。真值碰撞单独统计以保证安全评估诚实。

典型结果（起点对世界一无所知 → 边探索边到达）：

| 指标 | 数值 |
|---|---|
| 传感器 | range 3.5 m, FOV 120° |
| 到达目标时间 | 12.16 s |
| **沿途发现障碍** | **14 / 23**（其余从未进入路径，符合“只感知所见”） |
| 重规划次数 | 59（其中**前端全重规划 3 次**，由新发现障碍触发） |
| RMS / 最大跟踪误差 | 0.070 m / 0.204 m |
| **执行轨迹 vs 真值世界 是否无碰撞** | **true（0 步碰撞）** |

**结论**：即使规划器**初始对环境一无所知**，依靠有限视场传感器增量建图 + 发现即重规划，
机器人全程 **0 次真值碰撞**到达目标——证明了在“真实感知（非真值）”下闭环的有效性。

**仍存在的局限与后续方向**
- 感知建图在 z=z0 水平面内（2.5D 代价地图式），障碍按全高立柱处理；3D 体素建图可作进一步扩展。
- 局部重规划不切换同伦类：热启动只做局部形变，封死通道时由前端全重规划兜底（已实现）。
- 机器人为二阶积分点质量：未含差速/四旋翼真实动力学与姿态环。
- 可进一步加入窄通道/迷宫以压力测试避障成功率。

## 10. 轨道 B：Ackermann 车辆模型（同一套算法，自行车运动学）

为验证算法不局限于质点模型，将整套 Fast-Planner / EGO-Planner 流程迁移到**车辆（自行车）模型**：
状态 `(x, y, θ, v)`，控制量 `(速度, 前轮转角 δ)`，运动学 `θ̇ = v·tanδ / L`。
车辆参数：轴距 L=1.5 m，最大转角 δ_max=35°（→ 最小转弯半径对应曲率上限 κ_max = tanδ/L ≈ **0.4668 m⁻¹**），
巡航 1.5 m/s。**可行性约束由“加速度上限”替换为“曲率上限”**，控制器换为 **Pure Pursuit 纯跟踪**。
起点 (-7,-6)、终点 (7,6)。车辆版用固定种子（12）+ 三档**确定性场景**（`scenario`），**无随机批量**，
故下表是三档场景**各跑一次**的实测全量，避障以“路径是否无碰撞”的逐场景判定给出（而非统计成功率——
统计成功率见 §8 的质点版批量评估）。

### 10.1 任务①：车辆版前端 `ackermann_kino_astar_demo`（规划时间 / 轨迹长度 / 避障）

| 场景 | **规划时间 (ms)** | 扩展节点数 | 路径点数 | **轨迹长度 (m)** | 轨迹时长 (s) | **无碰撞** |
|---|---|---|---|---|---|---|
| dense（默认） | 132.6 | 5206 | 33 | 19.200 | 12.80 | ✅ true |
| narrow | 154.1 | 7754 | 35 | 20.400 | 13.60 | ✅ true |
| clustered | 466.6 | 15472 | 37 | 21.600 | 14.40 | ✅ true |

- **避障成功率（前端，3 场景）：100% (3/3)**，均满足最小转弯半径（运动基元按 δ 采样天然可行驶）。
- 规划时间随障碍密度上升（dense 133 ms → clustered 467 ms），与扩展节点数（5.2k→15.5k）正相关。
- 轨迹长度 19.2–21.6 m，越拥挤绕行越长。

### 10.2 任务②：车辆版后端 `ackermann_bspline_demo`（B样条 + ESDF + **曲率约束**）

后端在 **2D ESDF** 上梯度优化，可行性项改为**曲率惩罚**（把超过 κ_max 的弯压回）。400 次迭代。

| 场景 | 优化耗时 (ms) | 平滑度 优化前→后 | 平滑度降幅 | 最大曲率 优化前→后 (限 0.4668) | 轨迹长度 (m) |
|---|---|---|---|---|---|
| dense | 1.4 | 0.1675 → 0.0009 | **≈186×** | 0.1186 → 0.1278 | 19.15 |
| narrow | 1.2 | 0.1392 → 0.0009 | **≈155×** | 0.1186 → 0.0520 | 20.44 |
| clustered | 1.3 | 0.2087 → 0.0009 | **≈232×** | 0.1186 → 0.0506 | 21.59 |

- **平滑度代价稳定下降约两个数量级**（155–232×），优化仅需 ~1.3 ms。
- **最大曲率全程 ≪ 上限 0.4668 m⁻¹**（最大 0.128，仅占上限 27%），即优化后轨迹对该车**始终可行驶**，转弯平缓。
- 轨迹长度与前端基本一致（优化只做平滑/避障形变，不显著改变里程）。

### 10.3 任务③：车辆版 EGO vs ESDF `ackermann_ego_demo`（计算效率对比）

同一前端路径上并行跑两种后端，唯一差别是碰撞梯度来源：

| 场景 | ESDF 构建 (ms) | ESDF 总 (ms) | EGO 总 (ms) | **加速比** | 平滑度 ESDF/EGO | 最大曲率 ESDF/EGO | 长度 (m) |
|---|---|---|---|---|---|---|---|
| dense | 11.6 | 12.4 | 10.0 | **1.24×** | 0.0009 / 0.0010 | 0.0803 / 0.1179 | 19.14 |
| narrow | 8.2 | 8.8 | 6.8 | **1.29×** | 0.0010 / 0.0010 | 0.0536 / 0.0562 | 20.4 |
| clustered | 14.6 | 15.2 | 12.3 | **1.23×** | 0.0009 / 0.0009 | 0.0543 / 0.0566 | 21.6 |

> ⚠️ **重要对照结论（诚实呈现）**：车辆版 EGO 加速比仅 **1.23–1.29×**，远小于质点版（§6 单图 52×、§8 批量均值 44×）。
> 原因不是算法差异，而是 **2D 距离场构建本就便宜**：车辆版 2D-ESDF 构建仅 8–15 ms，
> 而质点版 3D-ESDF 构建需 ~108 ms。EGO 省去的正是“构建全局 ESDF”这一步，其收益随**地图维度与分辨率**急剧放大——
> 3D 大图里 ESDF 构建是瓶颈（故 EGO 优势显著），2D 小图里这一步本就很小（故 EGO 优势有限）。
> 这恰恰从反面印证了 EGO-Planner 的适用边界：**地图越大越高维，免 ESDF 的收益越大。**
> 两种后端的平滑度/曲率/长度三项几乎一致，说明 EGO 在 2D 下同样**不损失轨迹质量**。

### 10.4 任务④：车辆版闭环 `ackermann_closed_loop_demo`（Pure Pursuit + 动态重规划）

闭环串联：盒障碍地图 → Ackermann Kino A* 前端 → EGO B样条后端 → 时间分配 → **Pure Pursuit 纯跟踪**，
默认开启动态障碍 + 每 0.25 s 滚动重规划（控制步长 20 ms）。三档场景实测：

| 场景 | 参考点数 | 规划轨迹时长 (s) | 滚动重规划频率 | 末态到终点距离 (m) | 是否触发到达判定 |
|---|---|---|---|---|---|
| dense | 421 | 16.45 | ~3.8 次/s | 0.60 | ❌ 否（停在 0.60 m 外） |
| narrow | 433 | 17.05 | ~3.8 次/s | 0.74 | ❌ 否（停在 0.74 m 外） |
| clustered | 457 | 18.01 | ~3.8 次/s | 0.31 | ❌ 否（差 0.01 m 未达阈值） |

**实测结论（如实呈现，含一处局限）**：
- 三档场景**前端+后端规划均成功**，车辆按 Pure Pursuit 沿规划路径稳定行驶，运行期间持续滚动重规划（约 3.8 次/秒）。
- 但**车辆最终停在距终点 0.31–0.74 m 处**，未跨过到达判定阈值（距终点 < 0.3 m **且** 速度 < 0.5 m/s），
  因此节点**始终未打印最终的 `CLOSED-LOOP REPORT`**（即无正式的到达时间 / RMS / 安全标志输出）。
- 根因：**Pure Pursuit 的末端收敛能力有限**——车辆模型受最小转弯半径与前视几何约束，最后约 0.5 m 的精准对位较难，
  在终点附近“绕停”而非精确驶入。这是该闭环当前的**已知局限**，非前/后端规划本身的问题。

> **指标口径说明（重要）**：该节点打印的 `err` 是车辆到**前视目标点**（look-ahead 距离 1.8 m）的距离，
> 是 Pure Pursuit 的固有量，**并非横向跟踪误差**——故其数值不能与质点版 §7 的横向 RMS（0.13 m）直接比较。
>
> **可改进项**：把到达阈值放宽到 ~0.5 m、或末端切换“低速对位”控制律（减小前视距离 + 限速逼近），即可让车辆正式到达并输出完整闭环指标。需要的话可在此基础上补一版。

#### 车辆版核心指标汇总（对应课程要求“规划时间 / 轨迹长度 / 避障成功率”）

| 核心指标 | 车辆版结果（3 档场景） | 出处 |
|---|---|---|
| **规划时间**（前端） | 132.6 / 154.1 / 466.6 ms（dense/narrow/clustered） | §10.1 |
| 后端优化耗时 | ~1.3 ms（ESDF）；EGO 6.8–12.3 ms | §10.2 / §10.3 |
| **轨迹长度** | 19.20 / 20.40 / 21.60 m | §10.1 |
| **避障成功率**（前端逐场景无碰撞） | **100% (3/3)** | §10.1 |
| 平滑度改善 | 155×–232× | §10.2 |
| 最大曲率 / 上限 | ≤ 0.128 / 0.4668 m⁻¹（占 27%，始终可行驶） | §10.2 |
| EGO/ESDF 加速比（2D） | 1.23–1.29× | §10.3 |
| 闭环到达 | 停在距终点 0.31–0.74 m 处（未跨到达阈值，见上） | §10.4 |

> **避障成功率口径**：车辆版为确定性场景，故按“逐场景路径是否无碰撞”给出（前端 3/3=100%）；
> 真正的**统计意义避障成功率（96.7%）来自 §8 质点版 30 组随机批量评估**，两者口径不同，不可混用。

## 11. 轨道 C：Gazebo 物理仿真 + SLAM 在线建图（`planner_gazebo_demo`）

这是最贴近真实部署的一条轨道，把规划器放进**带物理引擎的 Gazebo 整车仿真**，并用**真实 SLAM 地图**（而非解析障碍）驱动规划：

1. **整车模型**：URDF（xacro）描述 Ackermann 小车，含激光雷达、转向与驱动关节，
   由 `ros2_control` 的 `ackermann_steering_controller` 驱动。
2. **阶段 1 — 在线建图**：`slam.launch.py` 启动 Gazebo + 整车 + `slam_toolbox`（2D Gmapping），
   人工用 `ackermann_teleop`（键盘 w/s/a/d）遥控绕场，`slam_toolbox` 实时构建占据栅格，
   完成后用 `nav2_map_server map_saver_cli` 保存为 PGM+YAML。
3. **阶段 2 — 栅格地图规划**：`planner.launch.py` 加载保存的地图，运行三种规划器之一
   （`kino_astar` / `bspline` / `ego`），在 `nav_msgs/OccupancyGrid` 上规划并发布 `nav_msgs/Path`，
   RViz 用 Map / Path / TF / RobotModel 显示（不再用 Marker）。起点 (-7,-6)、终点 (7,6)。

**与轨道 A/B 的本质区别**：障碍来自**真实 SLAM 栅格地图**（含噪声、未闭合、分辨率 0.05 m），
碰撞检查在栅格上做（`map_loader.hpp` 手写解析 P5 PGM + YAML）；车辆运动由**物理引擎**积分，而非解析二阶/自行车模型。

> **运行依赖说明（重要）**：该包依赖 `gazebo_ros_pkgs`、`gazebo_ros2_control`、`ackermann_steering_controller`、
> `ackermann_msgs`、`slam_toolbox`、`nav2_map_server` 等二进制包。**本仓库不含这些依赖**，在未安装它们的环境中
> `colcon build` 会在 `find_package(ackermann_msgs)` 处失败。需先 `apt install`（命令见 README §3.5）方可编译运行；
> 轨道 A/B 不受影响，可独立编译。

## 12. 总结

1. 完整复现了 **Fast-Planner 前端（Kinodynamic A*）+ 真实 ESDF 梯度优化后端** 与 **EGO-Planner 无 ESDF 核心**。
2. 后端 ESDF 梯度优化使平滑度代价下降约 **135 倍**、最小障碍距离提升约 **90%**（0.286→0.543 m）、
   最大加速度由超限 6.4 压回逼近限制 2.36 m/s²，**平滑性 / 安全性 / 可行性三目标同步改善**且全程无碰撞。
3. EGO 的 ESDF-free 梯度方案在保持轨迹质量的同时，相对 ESDF 方案获得约 **40~50×** 的计算加速，
   验证了“无 ESDF 梯度规划的计算效率提升”这一核心论点。
4. 搭建了“感知-规划-控制”全链路闭环，并两次升级仿真环境：
   (a) **动态障碍 + 实时滚动重规划 + 噪声扰动**——37 次毫秒级局部重规划在干扰下 0 碰撞到达；
   (b) **有限视场传感器 + 增量占据/ESDF 建图**（感知不作弊）——初始一无所知、边探索边规划，
       沿途发现 14/23 障碍、3 次前端全重规划，0 次真值碰撞到达。
5. 批量评估给出可复现的核心指标：前端成功率 100%、避障成功率 96.7%、EGO 平均加速比约 44×。
6. **轨道 B（Ackermann 车辆）**：同一套算法迁移到自行车运动学 + 曲率约束 + Pure Pursuit 跟踪，
   前后端均工作正常（平滑度 ↓~190×、曲率全程远低于最小转弯半径上限）；并诚实地观察到
   **2D 小图下 EGO 加速比仅 1.27×**，从反面印证了“免 ESDF 的收益随地图维度/规模放大”这一结论。
7. **轨道 C（Gazebo + SLAM）**：搭建了 URDF 整车 + `ros2_control` + `slam_toolbox` 在线建图 +
   栅格地图规划的完整流水线，把规划器从解析障碍推进到**物理引擎 + 真实 SLAM 地图**，更贴近真实部署
   （需安装 Gazebo/SLAM 相关依赖后运行）。
